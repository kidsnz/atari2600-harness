# Technique — sound effects (frame-table SFX)

**Goal:** the standard game SFX vocabulary as **data**: each effect is a frame table
(2 bytes/frame: `AUDC<<4|AUDV`, `AUDF`) generated by `pkg/audio` helpers, played by a tiny
per-frame tick. Effects are numerically verifiable (exact register sequence per frame).

Demo: `roms/techniques/sfx_demo.asm` (fire cycles through 5 effects).
CI: `scenarios/sfx_demo.json` (14 register asserts across all 5 effects + audio golden).
Generators: `pkg/audio` `PitchSweep` / `NoiseBurst` / `Blip` / `Arpeggio` / `EmitSFX` (unit-tested).

## The five standard recipes

| effect | generator | character |
|---|---|---|
| laser | `PitchSweep(4, 4, 22, 12, 12)` | square, descending sweep |
| explosion | `NoiseBurst(8, 6, 15, 24)` | white noise, linear decay to 0 |
| pickup | `Arpeggio(4, []int{19,14,11}, 4, 10)` | rising 3-note arpeggio (≈G5→C6→E6) |
| bounce | `Blip(6, 10, 10, 4)` | short low blip |
| engine | `NoiseBurst(3, 28, 8, 30)` | low rumble, slow decay |

## The player (12 lines of 6502, runs in overscan)

State: `sfxOn` / `sfxIdx` / `sfxLen` / table pointer (zp). Per frame, if active:

```
ldy sfxIdx
lda (ptr),y        ; AUDC<<4 | AUDV
sta tmp / and #$0F / sta AUDV0
lda tmp / lsr ×4   / sta AUDC0
iny / lda (ptr),y  / sta AUDF0
iny / sty sfxIdx
cpy sfxLen → done? AUDV0=0, sfxOn=0
```

Triggering = set pointer + length + `sfxOn=1` (the demo edge-detects INPT4 D7; a game calls the
same from its event code). One channel is enough for SFX; pair with the music driver
(`sound-driver.md`) for music + SFX priority.

## Why frame tables (and not code per effect)
- **Verifiable**: the scenario asserts the exact AUDC/AUDF/AUDV at chosen frames — confirmed all
  5 recipes register-exact on first run, plus a chained audio-digest golden.
- **Composable**: new effects are one `EmitSFX(...)` call away; tables live in ROM data.
- **Cheap**: tick ≈ 40 cycles in overscan; 2 bytes × duration per effect in ROM.
